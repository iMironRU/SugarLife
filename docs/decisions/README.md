# Журнал решений

Архитектурные решения проекта с обоснованиями. Каждая запись фиксирует контекст, варианты, выбор и причину.

## Формат записи

Один файл на одно решение: `NNNN-краткое-название.md`. Нумерация сквозная.

Шаблон:

```markdown
# NNNN. Краткое название решения

Статус: предложено | принято | отменено | заменено NNNN
Дата: ГГГГ-ММ-ДД

## Контекст

Что побудило принять решение. Какая проблема или вопрос.

## Рассмотренные варианты

- A. Вариант с плюсами и минусами
- B. Вариант с плюсами и минусами

## Решение

Какой вариант выбран и почему.

## Последствия

Что меняется в проекте после принятия. Что становится возможным, что — невозможным.
```

## Ранние решения (зафиксированы в концепции)

Эти решения уже зафиксированы в [`../concept.md`](../concept.md) разделе «Что было решено по дороге». По мере необходимости они могут быть детализированы отдельными записями здесь — особенно если возникнет вопрос «почему именно так».
