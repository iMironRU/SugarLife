# CLAUDE.md

Контекст для работы Claude Code в этом репозитории.

## Что это

SugarLife / СладкаяЖизнь — единое мобильное приложение для людей с диабетом 1 типа. Объединяет мониторинг, расчёт болюсов, обучение через игру и автоматическое управление инсулинотерапией.

## Где документация

- `docs/concept.md` — концепция проекта, читай первой
- `docs/decisions/` — журнал архитектурных решений (заполняется по мере)

Концепция — источник истины. Если возникает противоречие, надо либо обновить концепцию (с явным решением и записью в журнал), либо скорректировать работу.

## Принципы (выжимка из концепции)

Полные обоснования — в концепции, здесь — для быстрого обращения:

**UX**
- Всё в одном приложении, никаких прыжков между экранами/программами за базовыми задачами
- Action mode (что делать сейчас) и Analysis mode (что было) — это разные экраны
- Один аватар CrankyPanky через все режимы

**Архитектура**
- Один код ядра на iOS и Android. UI может различаться, логика — нет.
- Платформенная консистентность для критических функций
- Нулевой ручной ввод того что система знает
- Проактивные вопросы вместо ретроактивных правок

**Данные**
- Один источник правды для IOB / COB / истории
- Локальное хранение в MVP, без облака и аккаунтов
- Мульти-профили в архитектуре с первого дня

## Текущая фаза: HTML-прототипы

UX валидируется в HTML до выбора нативного стека. Любой человек открывает в браузере, тыкает, даёт обратную связь. Это даёт раннюю проверку интерфейсов и откладывает решение о технологиях.

**Структура прототипов:**
- `prototypes/index.html` — навигационный хаб со списком всех экранов
- `prototypes/screens/*.html` — отдельные экраны
- `prototypes/styles.css` — токены дизайна (цвета, отступы, размеры)

**Принципы для прототипов:**
- Чистый HTML + Tailwind через CDN. Без сборки, без фреймворков.
- Тёмная тема по умолчанию (для ночных проверок без ослепления)
- Mobile-first: макеты разрабатываются под телефон (~380px шириной)
- Дизайн-токены через CSS-переменные — поменять одно место → изменения везде
- Каждый новый экран добавляется в `prototypes/index.html`

**Деплой:**
GitHub Actions автоматически деплоит `prototypes/` на GitHub Pages при пуше в `main`. Конфигурация в `.github/workflows/deploy.yml`.

## Что НЕ делать

- Не создавать монолитные экраны со всем сразу — разделяй action / analysis
- Не использовать ярких насыщенных цветов — палитра тёмной темы приглушённая
- Не делать ретроактивных правок UX-логики — если экран что-то предполагает (например, что пользователь ввёл еду), это должно быть проактивным шагом, а не правкой задним числом
- Не добавлять зависимости от сторонних сервисов в прототипы — они должны работать офлайн
